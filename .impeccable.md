# Design Context — HTML-Materialien

## Users
- **Primary**: Lehrer (Admin) — verwaltet Materialien, erstellt Prüfungen, trackt Fortschritte
- **Secondary**: Kollegen — laden Materialien hoch, nutzen Dashboard
- **Tertiary**: Schüler — öffnen Materialien, machen Prüfungen (nicht direkt im Dashboard)
- **Context**: Berufskolleg NRW, GPU/PBP/Englisch, Unterrichtsvorbereitung

## Brand Personality
- **Modern & Professionell** — wie ein SaaS-Tool, nicht wie eine Schul-Website
- **Sachlich & Verlässlich** — klare Hierarchie, sofortige Orientierung
- **Kompakt & Informationsdicht** — alles auf einen Blick, keine Verschwendung
- 3 Wörter: **präzise, durchdacht, leistungsstark**

## Aesthetic Direction
- **Visual tone**: Clean, dicht, datengetrieben — wie Linear, Notion oder Vercel Dashboard
- **Theme**: Light + Dark (System-Preference oder Toggle)
- **References**: Vercel Dashboard, Linear.app, Notion (Sidebar), GitHub (kompakte Tabellen)
- **Anti-References**: Bunte Schul-Websites, verspielte Icons, übermäßige Abstände

## Design Principles
1. **Dichte vor Dekoration** — Information zuerst, Weißraum gezielt eingesetzt
2. **Hierarchie durch Kontrast** — Farbe, Gewicht, Größe, nicht durch Verschachtelung
3. **Konsistenz durch System** — jedes Element folgt dem Token-System
4. **Dark Mode ist First-Class** — nicht einfach invertiert, sondern durchdacht
5. **Schnell scannbar** — Lehrer haben wenig Zeit, Interface muss sofort liefern
